# Magic Key Assistant One-Pager

Magic Key Assistant is a local-first operations harness for solo operators and very small teams.

It turns natural-language conversation, notes, and follow-up work into durable operational records such as actions, decisions, meetings, leads, and review state. The goal is not to replace leadership judgment. The goal is to make real work easier to recover, track, and review.

## Why It Exists

Small teams lose context faster than they lose effort.

What breaks is usually not intent. It is continuity:

- decisions lose their rationale
- commitments stay implied instead of assigned
- follow-up work slips during busy weeks
- teams spend time reconstructing state they already had once

## Product Shape

The product is designed around a few non-negotiable ideas:

- local-first by default
- web console first
- Discord optional
- continuity routines on by default
- accelerators optional

## What It Does

Magic Key Assistant helps a team:

- capture actions, decisions, meetings, leads, and source links from ordinary language
- retrieve answers from both documents and structured records
- surface overdue, stale, unowned, or unresolved work
- keep an operator-visible control plane for routing, auditing, and configuration

## What Makes It Different

It is not trying to be another generic AI chat wrapper.

The differentiator is the harness around the model:

- durable records instead of transient chat history
- confirmation and auditability around mutations
- continuity review instead of passive note accumulation
- operator control over local and optional cloud routing

## Optional Accelerators

For teams that want more than continuity alone, the product can enable accelerator behavior through Scout, Dreamer, Rainmaker, Steward, and Curator. Those are opt-in layers, not the baseline product requirement.

## The Test

The product is valuable if a small team can run it locally for a few weeks and answer yes to questions like:

- Did it preserve context we would otherwise have lost?
- Did it make follow-through easier to see?
- Did it become part of our operating rhythm instead of another dashboard we ignore?
